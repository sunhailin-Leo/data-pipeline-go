RocketMQ
===
## RocketMQ Source Configuration Documentation
This configuration defines a RocketMQ source, used to consume messages from RocketMQ topics. Here is the configuration example for a RocketMQ source:
```json
{
  "source": [
    {
      "type": "RocketMQ",
      "source_name": "rocketmq-1",
      "rocketmq": {
        "address": "<RocketMQ Hosts>",
        "group": "<RocketMQ Group>",
        "topic": "<RocketMQ Topic>",
        "access_key": "<RocketMQ Access Key>",
        "secret_key": "<RocketMQ Secret Key>"
      }
    }
  ]
}
```

## Configuration Parameter Description
* **type：**
Specifies the type of the source, here it is "RocketMQ", indicating that RocketMQ is used as the data source.

* **source_name：**
A custom name for the RocketMQ source, which can be used to identify this source. In this example, it is set to "rocketmq-1".

* **rocketmq：**
Contains the core configuration information for the RocketMQ source:

  * **address：**

    The address of the RocketMQ NameServer or broker cluster, typically in the format "host:port". For example, "rocketmq.example.com:9876" specifies the address of the RocketMQ cluster.

  * **group：**

    The RocketMQ consumer group name. Different consumer groups can independently consume the same topic, and consumers within a group share the message consumption load. Example: "my_consumer_group".

  * **topic：**

    The RocketMQ topic name. Defines the topic where the messages to be consumed are located. Example: "my_topic".

  * **access_key：**

    The access key used to access RocketMQ, typically used for authentication and authorization. This key can be generated by the RocketMQ management platform.

  * **secret_key：**

    The security key used in conjunction with the access_key, ensuring the security of message transmission and permission control.

## Configuration Example
```json
{
  "source": [
    {
      "type": "RocketMQ",
      "source_name": "rocketmq-1",
      "rocketmq": {
        "address": "rocketmq.example.com:9876",
        "group": "my_consumer_group",
        "topic": "my_topic",
        "access_key": "my_access_key",
        "secret_key": "my_secret_key"
      }
    }
  ]
}
```

In this example:
* "rocketmq-1" is the defined RocketMQ source name.
* "address" specifies the RocketMQ cluster address "rocketmq.example.com:9876".
* "group" defines the consumer group "my_consumer_group", responsible for handling messages received from the specified topic.
* "topic" specifies the RocketMQ topic "my_topic" to be consumed.
* "access_key" and "secret_key" are credentials used to access and protect the RocketMQ service.
